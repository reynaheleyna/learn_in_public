# Microsoft Fabric Analytics Engineer

## Learning Goals

- Understand the Microsoft Fabric ecosystem and core workloads.
- Build end-to-end analytics solutions using OneLake, Lakehouse, Warehouse, and Power BI.
- Learn data ingestion, transformation, modeling, governance, and monitoring.
- Prepare for practical analytics engineering work and `DP-600`-aligned topics.

## Prerequisites

- Basic SQL skills
- Basic Power BI knowledge
- Familiarity with data warehousing concepts
- General understanding of Azure or Microsoft data services

## Learning Outline

### 1. Microsoft Fabric Fundamentals
- What Microsoft Fabric is
- Key workloads: Data Factory, Data Engineering, Data Science, Data Warehouse, Real-Time Intelligence, Power BI
- Understanding capacity, workspaces, and tenant basics
- Navigating the Fabric portal

### 2. OneLake and Data Storage
- What OneLake is and why it matters
- Shortcuts and centralized data access
- Delta/Parquet concepts
- Organizing data for analytics projects

### 3. Lakehouse and Data Engineering
- Creating and using a Lakehouse
- Uploading and managing files/tables
- Using notebooks for exploration and transformation
- Introduction to Spark in Fabric

### 4. Data Ingestion and Orchestration
- Using pipelines and dataflows
- Connecting to source systems
- Scheduling and monitoring refresh jobs
- Building repeatable ETL/ELT workflows

### 5. Data Warehouse in Fabric
- Creating a Fabric Warehouse
- Loading structured data
- Writing analytical SQL queries
- Designing fact and dimension tables

### 6. Semantic Models and Power BI
- Building semantic models
- Creating measures with DAX
- Designing reports and dashboards
- Applying row-level security basics

### 7. Real-Time and Event-Based Analytics
- Intro to event streams and real-time scenarios
- Streaming ingestion concepts
- Monitoring and near real-time reporting use cases

### 8. Governance, Security, and Administration
- Workspace roles and permissions
- Data sensitivity and governance basics
- Monitoring usage and performance
- Deployment considerations

### 9. Optimization and Best Practices
- Choosing Lakehouse vs Warehouse
- Performance tuning basics
- Organizing medallion architecture (`Bronze`, `Silver`, `Gold`)
- Cost and capacity awareness

## Suggested 4-Week Study Plan

### Week 1 — Foundations
- Learn Fabric architecture and workloads
- Explore OneLake, workspaces, and capacities
- Create your first Lakehouse

### Week 2 — Data Engineering
- Practice notebooks, Spark, and transformations
- Build a pipeline to ingest sample data
- Store curated data in Delta tables

### Week 3 — Warehousing and BI
- Create a Warehouse
- Model data into facts and dimensions
- Build a Power BI report on top of the semantic model

### Week 4 — Governance and End-to-End Project
- Add security and workspace permissions
- Monitor refresh/performance
- Build a small end-to-end project from ingestion to dashboard

## Hands-On Project Ideas

- Sales analytics pipeline from CSV to dashboard
- Finance reporting with Lakehouse + Warehouse
- Real-time operations dashboard
- Medallion architecture demo using sample datasets

By the end of this outline, you should be able to design and deliver a simple analytics solution in Microsoft Fabric from ingestion to reporting.
